Modernizing developer documentation

I built an online developer portal with totally reimagined technical documentation to replace legacy PDFs.

The problem

A major provider of geolocation solutions and identity verification had a reputation for abysmal documentation.

  • All docs were PDFs. There were dozens of documents, delivered in various ways, maintained by various teams.
  • Docs were often out-of-date, incorrect, or confusing. 10%+ of all customer issues were caused by bad documentation.
  • There were few code samples and instructions were unclear. A dozen specialists guided customers through implementations that lasted for months.
  • Customers made the same implementation mistakes over and over.
  • Customers complained that the documentation was bad. Competitors cited documentation quality as one of their differentiators.

I was hired to build the company's first technical documentation team, take ownership of the docs and the process, and build a new, modern developer portal.

The solution

The company needed a modern developer portal with better content, good navigation design, and effective search. I designed a modern docs-as-code CI/CD tech and process stack built on lightweight, popular open-source tools and cloud technology. This ensured the system would be modern, highly automated, and familiar to developers who already used the same workflow for software development.
